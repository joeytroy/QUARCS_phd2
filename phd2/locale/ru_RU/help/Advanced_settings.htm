Настройки

Настройки вызываются нажатием кнопки 'Мозг' в главном окне.  PHD2 имеет большой набор настроек для оптимизации гидирования,  разобраться в которых довольно просто.   Для всех полей есть подсказки, маленькие окошечки с детальными описаниями каждого поля. Чтобы увидеть подсказку для поля, нужно просто поместить курсор на него.  Зачастую этой информации хватает для понимания.   Из-за обилия настроек они размещены по вкладкам, которые могут быть открыты по щелчку левой кнопки мыши по их названиям. В нижней части каждой из вкладок находятся кнопки 'Принять' и 'Отменить'. При нажатии кнопки 'Принять' сделанные  в любой из вкладок изменения вступят в силу; нажатием кнопки 'Отменить' изменения будут отменены.

Общее



Здесь представлены общие настройки приложения. 

Основные настройки

Остальные настройки в закладке 'Общие' хорошо описаны своими подсказками, но для полноты приводятся и здесь:
  • 'Язык' - определяет язык, используемый в PHD2.  Изменение языка обычно требует перезагрузки программы
  • 'Сброс конфигурации' - сбрасывает все настройки к их начальным на момент установки PHD2 значениям
  • 'Сброс сообщений "Больше не показывать"' - разрешает вывод предупредительных сообщений если они были запрещены ранее
  • 'Обновление ПО'
    • Проверять обновления автоматически - позволить PHD2 автоматически проверять обновления при запуске программы
    • Проверять только основные выпуски - определяет, учитывать ли разработческие сборки при проверке обновлений
    Подробнее см. в разделе Обновление ПО.
  • 'Формат логгирования изображений' - задаёт формат файлов при логгировании звёздных изображений
  • 'Расположение лог-файла' - определяет папку, где будут храниться лог-файлы гидирования и отладки PHD2. В Windows по умолчанию лог пишется в файл, расположенный в папке "Мои Документы" ("My Documents"), связанной с текущим пользователем..
  • Настройки подвижек
    • 'Случайный режим' - для определения величины и направления подвижки используется генератор случайных чисел, с учётом ограничений, налагаемых режимами подвижек только по прямому восхождению или при гидировании только по прямому восхождению.
    • 'Режим спирали' - спиральные подвижки фиксированной величины по часовой стрелке.  Хороший выбор при значительном фиксированном шуме или проблемах с люфтом склонения
    • 'Делать подвижки только по RA' - для использующего сервер PHD2 внешнего приложения разрешает подвижки только по оси прямого восхождения
    • 'Масштаб подвижек' - необязательный параметр, определяющий максимальное ограничение для подвижек. См. Операции подвижек
  • 'Включить диагностическое логгирование изображений' - используется главным образом для поддержки продукта и диагностирования проблем с определением звёзд и измерениями, но также может применяться для просмотра гидирующих изображений и с другими целями.  Полученные при гидировании кадры сохраняются в формате FITs согласно заданным фильтрам в подпапках папки логгирования, где время и дата съёмки являются частью имени подпапки.  Имена файлов содержат время съёмки и причину сохранения кадра.  В заголовке файла формата FITs содержится полезная информация, такая как время экспозиции.  Поскольку логи используются преимущественно при решении возникших проблем, они автоматически удаляются через 30 дней.  Чтобы их сохранить, нужно переименовать папки или перенести их в другое место.  При логгировании по наступлении одного из "событий", например, потери звезды или серьёзных ошибок, будет сохранён ряд изображений перед и после наступления этого события. Ниже описаны события и типы фильтров:
    • 'Для всех кадров с потерянной звездой' - сохраняет все изображения для событий, связанных с потерей звезды, несмотря от того, чем она вызвана (низкий уровень сигнал/шум, изменение яркости и прочее)
    • 'Для всех кадров автовыбора звезды' - записывает кадры, использовавшиемся для автоматического выбора звезды независимо от результата, то есть даже если выбор не удался.
    • 'Когда превышен порог относительной ошибки' - записывает изображения при отклонении звезды на величину, большую средней ошибки.  Например, если среднеквадратичная ошибка равна 0.5 пикселей, а ошибка текущего кадра составляет 1.5 пикселя, то средняя ошибка равна 3.
    • 'Когда превышен порог абсолютной ошибки' - записывает изображения при отклонении звезды более заданного количества пикселей.
    • 'Пока не достигнуто заданное колиество итераций'  - записывает изображения до тех пор, пока не превысится заданное число итераций.  После этого счётчик будет сброшен в ноль.
Поскольку изображения сохраняются в стандартном формате, их можно посмотреть и проанализировать с помощью множества программ, немалая часть из которых бесплатна.  Большинство программ для работы с изображениями могут взаимодействовать с другими, более специфическими средствами измерения звёзд и оптического качества поля зрения.  В Интернете можно найти программы для работы с форматом FITs для любой используемой вами платформы.  Для того, чтобы проверить фокус или качество полученного с камеры изображения, достаточно PHD2.  Когда PHD2 не занята серийной съёмкой или гидированием, перетащите сохранённое в формате FITs изображение на главное окно и вы увидите его там.  Для этого не нужно подключаться к какому-либо оборудованию.  Можно настроить яркость, выбрать звезду (вручную или автоматически) и использовать средство Профиль Звезды для просмотра её данных.

Камера



Используются следующие настройки камеры:
  • 'Уменьшение шума' - задаёт алгоритм уменьшения шумов камеры, если недостаточно использования темновых кадров.  Возможные варианты: Нет, 2x2 среднее и 3x3 медиана. Оба алгоритма значительно уменьшают шум. 3x3 медиана особенно эффективен при удалении горячих пикселей и не оказывает существенного влияния на точность гидирования.  Однако,  создание карты битых пикселей более эффективно и меньше влияет на способность распознавания слабых звёзд.
  • 'Частота замедленной видеосъёмки' - задаёт задержку между гидирующими кадрами.  Это может быть полезно, когда кадры очень короткие, чтобы не перегружать камеру и монтировку передачей большого объёма данных.
  • 'Автоэкспозиция' - задаются настройки управления временем автоэкспозиции.
    • Минимальная экспозиция - минимальное время выдержки. PHD2 не будет использовать более короткие выдержки, даже если уровень сигнал/шум гидируемой звезды выше, чем заданный. Если установлено слишком короткое время минимальной экспозиции, слишком маленькие корректировки не позволят добиться наилучших результатов при гидировании. Пользователям с адаптивной оптикой могут понадобиться экспозиции покороче, поскольку быстрые короткие поправки зачастую хороши с адаптивной оптикой.
    • Максимальная экспозиция - максимальное время выдержки. До тех пор, пока звезда для гидирования не будет выбрана, PHD2 будет использовать максимальные выдержки. После того, как звезда будет выбрана, PHD2 будет постепенно уменьшать выдержку пока желаемый уровень сигнал/шум не будет достигнут.
    • Сигнал/шум звезды - среднее значение уровня сигнал/шум, которое PHD2 будет пытаться достичь, изменяя время экспозиции. Соотношение сигнал/шум может изменяться от кадра к кадру даже с одинаковыми выдержками, что надо учитывать при выборе его значения. PHD2 отвергает кадры, где сигнал/шум ниже 3.0. Значение по умолчанию 6.0 обеспечивает достаточную устойчивость к колебаниям ниже уровня 3.0, но желательно использовать значение побольше.
  • 'Размер пикселей' - Размер пикселей гидирующей камеры в микронах. Это один из двух параметров, на основании которых PHD2 вычисляет масштаб изображения и выводит статистику в угловых секундах (другой - фокусное расстояние гидирующего телескопа - задаётся во вкладке 'Гидирование').  Значение этого параметра можно узнать из документации к камере. В случае неквадратных пикселей можно выбрать одно из измерений или среднее между ними. Эта величина не влияет на точность гидирования, поэтому небольшая неточность не повлечёт каких-либо проблем. При использовании биннинга нужно использовать оригинальный размер пикселя. Замечание: изменение параметра может быть недоступно если драйвер камеры может сообщить PHD2 размер пикселя. В этом случае вы увидите полученное от драйвера значение. При задании коэффициента биннинга на камере, а не в PHD2 прочитанный размер пикселя может измениться тоже. Поэтому следует задавать биннинг в PHD2 ( см. ниже).
  • 'Чувствительность камеры' - Устанавливает уровень чувствительности для камер, поддерживающих эту возможность. Если вы хотите гидировать по яркой звезде с длинными выдержками или изображение очень шумное, уменьшите значение этого параметра. Промежуток значений и их эффект зависят от конкретной камеры, поэтому  PHD2 определяет этот параметр как процентное отношение парамерта между минимальной и максимальной чувствительностями камеры. Например, если абсолютная чувствительность камеры меняется от 40 до 80, используемое PHD2 значение 50 % будет означать чувствительность камеры 60.
  • 'Отключить не отвечающую камеру через (сек) - Иногда связь с камерой может сбоить, зачастую из-за проблем с соединением по USB.   Камера не возвращает запрошенные данные и PHD2 зависает.  Параметр определяет как долго PHD2 будет ждать ответа после того, как ожидаемое время экспозиции истечёт.  Например, таймаут в 5 секунд в сочетании с 2-секундной экспозицией даст время 7 секунд, в течение которых PHD2 будет ожидать ответа.  Если он за это время не будет получен, PHD2 попытается прервать операцию, отключит камеру и вывести в главное окно сообщение об ошибке.  Поскольку проблемы с оборудованием часто происходят на аппаратном уровне, эта попытка не всегда бывает успешной.  Во избежание ложных срабатываний лучше выставить величину побольше.  Кроме того, если гидирующая камера подключена к основной и управляется ею, значение параметра должно быть больше ожидаемого должно быть больше максимально ожидаемого времени загрузки полнокадрового изображения с основной камеры.  В настоящее время это относится к пользователям драйвера  SBIG, идущего в комплекте с программным обеспечением Sequence Generator Pro.  Независимо от того, насколько успешно PHD2 способен обработать эту ситуацию, проблема с аппаратурой или драйвером камеры должна быть разрешена до того, как гидирование будет возобновлено.
  • Биннинг - для камер, поддерживающих аппаратный биннинг можно задать его значение.  Более детально это рассматривается дальше. Этот параметр появится только если камера подключена к PHD2 и поддерживает аппаратный биннинг.
  • 'Использовать подкадры' - Камеры, поддерживающие эту возможностьPHD2 будут передавать только подкадр каждой экспозиции. Это очень полезно для камер с низкой скоростью передачи данных. Когда этот параметер задан, после выбора звезды передаются только маленькие подкадры. Это относится и к калибровке и к гидированию.  Пока звезда ещё не выбрана, передаётся полный кадр, но сразу после её выбора передаются маленькие подкадры.   Если вы используете подкадры, но хотите использовать полный кадр, чтобы выбрать другую звезду, при нажатой клавише shift щёлкните где-нибудь в пределах окна.
Использование биннинга
Некоторые из камер поддерживают аппаратный биннинг, который может быть полезным при больших фокусных расстояниях или очень маленьких пикселях гидирующей камеры. Такое часто бывает при гидировании по тусклым звёздам; размывание изображения слабой звезды не даёт преимуществ, поскольку может привести к низкому значению коэффициента сигнал/шум (SNR).  Используя биннинг, вы можете уменьшить влияния шума считывания камеры и тем самым улучшить соотношение сигнал/шум.  Выбор значения биннинга больше единицы приведёт к следующим последствиям:
  1. Изображения звёзд будут иметь больший коэффициент сигнал/шум и их будет легче выделить из фонового шума.  Может быть особенно полезно если у вас в наличии только слабые звёзды (те, у которых коэффициент сигнал/шум ниже 10).
  2. Объём получаемых с камеры данных уменьшится пропорционально квадрату коэффициента биннинга.  Может быть полезным если камера интенсивно использует USB - даже если опорная звезда достаточно ярка для выделения и без биннинга.  Разумеется, использованием подкадров после выбора звезды можно достичь того же эффекта.
  3. Разрешение изображения гидирующей камеры уменьшится на коэффициент биннинга.  Вряд ли будет проблемой при масштабе изображения без биннинга меньшим 1 угл.сек/пикс, но при значениях, больших, чем 3, возможны проблемы с гидированием.  Возможно, понадобятся эксперименты, поскольку результаты гидирования зависят и от масштаба изображения основной камеры.
Каждый уровень биннинга требует своих темновой библиотеки и карты дефектных пикселей - они не взаимозаменяемы и не могут быть преобразованы автоматически.  Если вы хотите изменять значения биннинга, создайте отдельный профиль для каждого из них со своими темновой библиотекой и картой дефектных пикселей.  После этого значения биннинга можно менять простым выбором соответствующего профиля. Чтобы проверить корректность биннинга, можно воспользоваться окном Статистики, показывающим размер кадра и текущие настройки биннинга камеры.

Гидирование



Здесь задаются параметры калибровки, слежения за звездой и поведение гидирования, общее для всех алгоритмов..

Слежение за опорной звездой
  • 'Область поиска' - определяет размер прямоугольника поиска в пикселях.  Возможно, это значение придётся увеличить если монтировка работает не очень хорошо или, что более вероятно, недостаточно точно установлена на небесный полюс.  Возможно вы также захотите временно увеличить его при использовании Помощника Гидирования, чтобы при измерении люфта опорная звезда не оказалась потеряна.  Помните, что чрезмерное увеличение области поиска может привести к тому, что внутри её окажутся несколько звёзд, что может привести к проблемам гидирования.
  • 'Определение звёздной величины' - заставляет PHD2 следить за яркостью и размером гидируемой звезды в сравнении с небесным фоном.  Вы можете выключить определение звёздной величины или же установить её в максимальное значение, если при мелком масштабе изображения часто теряете звезду. 
  • 'Допуск изменения звёздной величины' - если установлен параметр 'Определение звёздной величины', PHD2 выдаст ошибку 'звезда потеряна', если яркость и размер звезды изменятся больше, чем на заданный процент.  Это может быть полезным если в области отслеживания находятся две звезды, чтобы ошибочно не переключиться между ними.  Также это предотвращает ошибки, вызванные тонкими облаками, большим уровнем шума камеры или распадом альфа-частиц, но не будет надёжным при гидировании по слабой звезде.  Если ошибок потери звезды слишком много при том, что звёзды ясно видны на экране, попробуйте увеличить значение этого параметра.  Установка его в 100 убирает предупреждения полностью.
  • 'Минимальный HFD звезды в пикселях' - задаёт минимальный кружок, куда падает половина излучения подходящей опорной звезды. Используется для предотвращения захвата горячих пикселей. Подобрать его можно, выбрав слабую звезду, в которой вы уверены, что это не пиксель и посмотреть HFD в её профиле. Используйте этот параметр, если PHD2 попытается гидировать по горячему пикселю.
Калибровка
  • 'Фокусное расстояние' - фокусное расстояние гидирующего телескопа в миллиметрах. Этот параметр нужен для подсчёта PHD2 масштаба изображения и определения производительности гидирования в угловых секундах.  Другой необходимый для этого параметр определяет размер сенсора гидирующей камеры в пикселях и находится в закладке 'Камера'. Если их значения установлены неправильно, параметры гидирования по умолчанию могут оказаться неверными, что скажется на его качестве. Если профиль был создан Помощником Нового Профиля, эти значения, скорее всего, правильны.
  • 'Шаг калибровки' -  определяет продолжительность гидирующих импульсов, которые PHD2 будет использовать при калибровке.  Использование этого параметра описано в секции 'Автоматическая калибровка' справочной страницы 'Основы использования'.  Значение можно изменять, когда звезда перемещается во время калибровки слишком быстро или, наоборот, слишком медленно.   Лучше проводить калибровку в области до 30 градусов от небесного экватора с со значением параметра, обеспечивающим 8-14 калибровочных шагов в каждом направлении.   Кнопка 'Дополнительные' справа от поля ввода запускает диалоговое окно, которое помогает вычислить подходящее значение (см. ниже)
  • 'Автовосстановление калибровки' - автоматическая загрузка последних калибровочных данных при подключении оборудования.  При использовании соединений ASCOM или Indi или работе с 'дополнительной' монтировкой этот параметр будет установлен автоматически при подключении.  И наоборот, если у PHD2 нет необходимой информации, параметр будет сброшен автоматически.  Помощник нового оборудования спросит, использовать ли эту возможность. Учтите, что автовосстановление работает для каждого профиля отдельно и произойдёт только после выбора профиля и подключения к оборудованию. Если вы перед началом сессии гидирования хотите перекалиброваться, то надо сбросить калибровку монтировки (см. ниже). Используя этот параметр, вы должны быть уверены, что гидирующая камера не поворачивалась относительно телескопа или неба. 
  • ''Считать Dec ортогональным RA' - Обычно при калибровочном процессе определение углов камеры для прямого восхождения и склонения происходит независимо.  При этом большая точность не нужна и поведение по умолчанию обычно работает хорошо.  Однако, если у монтировки очень большая периодическая ошибка или турбулентность очень велика, может понадобиться указать явно перпендикулярность углов прямого восхождения и склонения.  Если выбран этот параметр, PHD2 подсчитает угол камеры для прямого восхождения, затем примет угол склонения перпендикулярным ему.
  • 'Сброс калибровки' - PHD2  очистит калибровочные данные и проведёт повторную калибровку перед началом гидирования.   Для этого могут быть разные причины: перемещение гидирующей камеры, перекладка после прохождения меридиана и т.д.  То же самое достигается нажатием Shift-Click на кнопк гидирования, расположенной на главном экране, что приведёт к повторной калибровке.
    • 'Использовать компенсацию склонения' - если PHD2 получает информацию о координатах наведения монтировки с помощью соединения ASCOM  (основного или дополнительного), она автоматически рассчитает величину гидирования по прямому восхождению на основе текущего склонения.  Деактивировать этот параметр следует лишь для особых случаев, например, когда контроллеры монтировки SiTech применяют автоматическую компенсацию. Не спутайте этот параметр с 'Компенсацией люфта склонения', это совершенно другое.
Общие параметры гидирования
  • ''Быстрая центровка после калибровки или подвижек' - во время калибровки или подвижек монтировка может быть перемещена от начальной позиции.  Если выбран этот параметр, PHD2 переместит монтировку   назад к выбранной позиции так быстро, как позволяет параметр алгоритма гидирования 'Максимальная длительность'.  Поскольку это лишь оптимизация, использовать этот параметр необязательно.  Если в результате быстрой центровки звезда теряется, его нужно выключить.  Такое бывает при большом отклонении полярной или значительной периодической ошибке прямого восхождения. Помощник Гидирования может помочь разобраться в сути проблемы.
  • 'Инвертировать импульсы по склонению после перекладывания меридиана' - указывает PHD2 что делать с калибровочными данными когда используется функция "переложить" их.  Некоторые новые монтировки отслеживают это и автоматически меняют направление гидирующих импульсов по склонению, старые же не делают этого.  В любом случае PHD2 нужно знать поведение монтировки.  Эту информацию бывает нелегко найти, поэтому у PHD2 для автоматического определения правильных значений есть Калибровка перекладки меридиана. Кроме этого, можно устроить небольшой эксперимент:
    • Не выбирая этот параметр, откалиброваться с одной стороны меридиана, затем навести монтировку в другую сторону от него.
    • При работе через ASCOM или Indi или при использовании "дополнительной" монтировки начните гидирование.
    • При гидировании через ST-4 когда PHD2 не знает координат нужно сначала выбрать  'Переложить откалиброванные данные' в меню 'Инструменты -> Изменить калибровку' и начать гидирование.
    • Если гидирование проходит нормально, оставить параметр невыбранным; но
    • если склонение собъётся по склонению, выбрать параметр и повторить всю процедуру, включая калибровку.
  • 'Включить гидирующие импульсы' - обычно этот параметр включен, поскольку он заставляет PHD2 посылать гидирующие команды монтировке.  Однако, иногда может потребоваться отключить их. Обычно это делают, чтобы пронаблюдать некорректное поведение монтировки, например, график периодической ошибки или недостаточно точно установленный полюс.В большинстве случаев проще запустить Помощник Гидирования, который автоматически установит этот параметр.
  • 'Остановить гидирование при движении монтировки' - при гидировании через интерфейс ASCOM PHD2 может определить некорректное движение монтировки и прекратит посылать гидирующие команды. Параметр следует включить, если у драйвера монтировки бывают такие проблемы. Обычно наведение с пульта не распознаётся. В любом случае PHD2 будет продолжать пытаться отслеживать опорную звезду, поэтому обычным результатом будет потеря звезды, о чём PHD2 просигналит звуком и визуально.
Параметры калибровки.



Для обзора и изменения калибровочных параметров четыре верхних поля должны быть корректно заполнены.  Если фокусное расстояние и размер пикселей камеры уже были определены Помощником Нового Оборудования или введены в закладках 'Общие' и 'Камера, эти значения появятся и тут.  При использовании соединения ASCOM поля "Скорость гидирования" и "Калибровка по склонению" обычно также уже будут корректно заполнены, если драйвер уже не передаст их правильные значения.  В противном случае их придётся заполнять самостоятельно.  Скорость гидирования задаётся кратной солнечной - большинство монтировок используют что-нибудь вроде 1X or 0.5X солнечной, но можно задать и что-нибудь другое.   Важно: изменение значения скорости гидирования никогда не изменит скорости гидирования монтировки - её можно поменять только с пульта через драйвер монтировки.   Если это поле уже заполнено, его изменение не даст нужного эффекта. Значение параметра 'Шаги калибровки' можно оставить равным 12, что обычно подходит для хорошей калибровки.  Использование значительно меньших величин увеличивает вероятность того, что турбулентность и небольшие погрешности монтировки приведут к ошибкам калибровки.  Также можно изменить полное расстояние калибровки если используемая по умолчанию величина 25 пикселей неадекватна. Будьте аккуратны, поскольку "больше" не значит "лучше". При большом расстоянии калибровка сильнее подвержена большим неисправляемым периодическим ошибкам оси прямого восхождения и большому люфту оси склонения. При изменении значений в этих полях  PHD2 пересчитает масштаб изображения и шаг калибровки.   Если затем кликнуть 'Принять', эта величина попадёт в поле 'Шаг калибровки' вкладки 'Монтировка'.  При этом фокусное расстояние и размер пикселей запишутся в соответствующие поля во вкладках 'Общие' и 'Камера', так что сделанные в калькуляторе изменения будут отражены и там.   При нажатии кнопки 'Отменить' этого не произойдёт. Опять-таки, PHD2 никогда не меняет настройки скорости гидирования монтировки, независимо от того, что введено в поле 'Скорость гидирования'.

Целью подсчёта является размер шага, обеспечивающий точное гидирование без потери времени; значение, определённое автоматически, обычно хорошо подходит. Если калибровка проходит успешно без сообщений об ошибках, следует оставить в покое её параметры. При использовании Помощника Нового Профиля менять их приходится редко.

Алгоритмы



Эта вкладка определяет алгоритмы гидирования и их параметры.  Параметры отображаются в соответствии с выбранным алгоритмом, поэтому все относящиеся к алгоритмам гидирования параметры будут рассмотрены в отдельном разделе.

Оставшиеся независимые от алгоритма гидирования параметры описаны ниже:  
  • 'Наибольшее время по α' - определяет максимальную продолжительность гидирующего импульса для прямого восхождения.  Его можно задать ниже величины по умолчанию, чтобы избежать больших отклонений, вызванных ложными событиями (например, внезапным порывом ветра или появлением горячего пикселя). Однако, если вы защитили себя от горячих пикселей (минимальный HFD звезды), менять этот параметр приходится редко.
  • 'Использовать компенсацию люфта' - определяет, будет ли PHD2 компенсировать люфт при изменении направления гидирования по склонению.  Измерение люфта и вычисление подходящего начального значения для компенсации производится Помощником Гидирования. Значения дополнительных компенсирующих импульсов показаны в поле 'Количество' рядом с  флажком. Это значение PHD2 может изменять в обе стороны, чтобы улучшить результаты гидирования. В любом случае настройка проводится консервативно, чтобы избежать нестабильности гидирования. Поскольку PHD2 умеет определять и исправлять избыточные поправки, её компенсация люфта должна работать лучше, чем у контроллеров большинства монтировок. Используя исправление люфта с помощью PHD2, необходимо отключить исправление люфта контроллером монтировки. Подробнее см. раздел Помощник Гидирования. Параметр 'Наибольшее время по δ' может быть настроен автоматически дабы избежать укорачивания импульса исправления люфта. Не следует ожидать хорошего исправления если длительность люфта составляет много секунд..
  • "Наибольшее время по δ' - определяет максимальную продолжительность гидирующего импульса для склонения (по аналогии с аналогичным параметром для α).  Это может быть полезным, чтобы избежать избыточной корректировки с последующим переключением направления при гидировании по склонению.
  • 'Режим гидирования по склонению' - предоставляет дополнительную возможность управления гидированием по склонению.  В отличие от гидирования по прямому восхождению, ошибки гидирования по склонению не вызваны несовершенством механизма монтировки.  Причиной их в основном являются ошибка выставления полярной или прогиб.  Ошибка должна быть гладкой и, как правило, однонаправленной, чтобы не было избыточных коррекций.  Режим по умолчанию 'Авто'  допускает смену направлений коррекции, однако, если у вашей монтировки есть значительный люфт, вы можете захотеть запретить смену направлений вовсе.  В этом случае вы можете выбрать режимы 'север' или 'юг', и корректировки будут ограничены выбранным направлением.  Следует иметь в виду, что после избыточной корректировки в этих режимах в результате избыточной корректировки настройка на опорную звезду может на продолжительное время оказаться сбитой. Чтобы избежать этого, при отключенной смене направлений корректировки следует применять консервативные параметры агрессивности.  И, наконец, режим 'Нет' совсем отключает гидирование по склонению, что подходит для монтировок, которые не поддерживают гидирования по склонению..
  • 'Сбросить' - сбрасывает параметры гидирования по прямому восхождению и склонению до значений по умолчанию.  Величина минимального движения будет установлена используя алгоритм, применённый помощником создания нового профиля. Если вы задавали с его помощью значение минимального движения, вам, вероятно, придётся это повторить.

Компенсация люфта склонения

У большинства популярных монтировок есть люфт оси склонения, из-за которого происходит задержка при смене направления гидирования.  Во время этой задержки ось склонения не реагирует на команды гидирования. У многих монтировок есть настройки компенсации люфта, однако, они применимы, в основном для визуальных наблюдений, когда большая точность не требуется и не должны использоваться при гидировании.  Поскольку величина компенсации в данный конкретный момент зависит от позиции наведения и нагрузки системы, фиксированное значение обычно приведёт к колебаниям, от которых невозможно будет избавиться.  Компенсация, используемая PHD2, является адаптивной, то есть зависит от результатов измерений.  Перед тем, как её включить, следует запустить Помощник Гидирования, чтобы измерить люфт - время задержки при смене направления движения по склонению.  Чем выше скорость гидирования (например, 0.9x солнечной), тем короче эта задержка будет.  Если её продолжительность не более 3 секунд, то Помощник Гидирования посоветует попробовать компенсацию люфта.  Если вы примете этот совет, настройки компенсации люфта будут заданы автоматически.  Элементы управления включают "минимальное" и "максимальное" значения компенсации.  Если вы хорошо разбираетесь в поведении своей монтировки, то можете задать эти значения вручную, в противном случае оставьте значения по умолчанию.  Алгоритм компенсации люфта обычно работает хорошо, если задержка при люфте не превышает нескольких секунд и у монтировки нет других серьёзных механических проблем.  Ожидайте короткого периода нестабильности поскольку изначальное состояние системы передачи оси склонения неизвестно - дайте ей стабилизироваться перед началом съёмок.  Если же стабилизация колебаний так и не наступила, выключите компенсацию и отправьте свой журнал отладки на форум поддержки PHD2.  Следует иметь в виду, значение компенсации люфта часто может изменяться в зависимости от координат наведения телескопа.  Это может быть вызвано неравномерным износом зубчатого колеса на оси склонения или различиями в нагрузке, приложенной к оси.

Однонаправленное гидирование по склонению

Как обсуждалось в другом месте, люфт склонения некоторых монтировок слишком велик для успешного гидирования в обоих направлениях: северном и южном.  Положение можно улучшить, гидируя по склонению лишь в одном направлении, поскольку гидирование по склонению исправляет медленный дрейф из-за ошибки выставления полярной и, в меньшей степени, механического изгиба.  Можно даже ухудшить настройку полярной по склонению, чтобы дрейф звезды в одном направлении был более явным и турбулентность реже уводила звезду в обратном направлении;  это не ухудшит качества гидирования, но может привести к вращению поля при съёмках вблизи полюса на камеру с большим сенсором.  Прежде чем приступить к однонаправленному гидированию, было бы хорошо выровнять полярную в пределах 5 - 10 угловых минут.  Сделайте пробный снимок на основную камеру участка с наибольшим склонением, которое вы собираетесь использовать для съёмок - например, 70 градусов к северу.  Если вращение поля не видно, оставьте настройку полярной как есть.  При любом отклонении от точной установки полярной направление поправок по склонению будет меняться на противоположное в некоторой небесной точке (на самом деле в двух, но одна из них обычно находится под горизонтом).  Их положение зависит от того, как сильно вы ушли от точной настройки полярной по азимуту и высоте.  Возможно даже, что точка смены направления гидирования окажется достаточно близко к горизонту, чтобы не брать её во внимание при съёмке.

Чтобы настроить однонаправленное гидирование, нужно сделать следующее:
  1. Выбрать подходящую для гидирования звезду и на несколько минут запустить Помощник Гидирования. Наблюдать за графиком, пока не станет ясно, уходит ли гидируемая звезда на север или на юг, после чего включить гидирование по склонению в обратную: например, если звезда уходит на север, надо выбрать режим гидирования 'юг'.
  2. Попробовать алгоритм 'Сглаживание2' для склонения с умеренным значением агрессивности, например 50%.  При большом уровне агрессивности звезда сместится и потом медленно вернётся в исходное положение, поэтому лучше сделать несколько небольших поправок, чем одну большую.
  3. Понаблюдать за графиком, чтобы удостовериться, что поправки осуществляются в нужном направлении и звезда не уходит из заданного положения.  Через некоторое время вы увидите, что дрейф уменьшается.  Это означает, что вы медленно приближаетесь к точке смены дрейфа на противоположный и должны быть готовы изменить направление поправок по склонению в соответствии с этим.
  4. Если вам нужны подвижки, можно попробовать делать их только по прямому восхождению, чтобы не нарушить гидирование по склонению.

    5. Другие устройства



    При использовании адаптивной оптики и/или поворотного устройства появится вкладка 'Другие устройства'. Верхний раздел посвящён устройству алаптивной оптики, если таковое подключено. Эти параметры могут быть использованы для управления калибровкой и операцией 'толчка'.  Поле 'шаг калибровки' задаёт PHD2 величину движения  наклонного элемента в каждом из направлений (вверх/вниз/вправо/влево), измеряемую в шагах AO во время калибровки.  Положение гидируемой звезды измеряется на каждом этапе калибровки и параметр 'образцы для усреднения' говорит PHD2 сколько образцов брать.  Усреднение изображений важно, поскольку из-за турбулентности изображение гидируемой звезды всегда чуть "прыгает". Как обсуждается в другом месте, устройство AO может вносить лишь ограниченные поправки в движение гидируемой звезды.  Чтобы начать коррекции толчками до того, как эти ограничения будут достигнуты, предназначен параметр 'процент толчка'.  Поле 'Смещение АО' задаёт количество шагов, которые АО может сделать по каждой оси. Для большинства устройств подходит значение по умолчанию, хотя в некоторых случаях оно может быть слишком большим. Если во время калибровки АО достигает ограничений, значение этого параметра можно уменьшить. Полная коррекция выполняется пошагово - поле 'шаг толчка' управляет этими приращениями.  Если гидируемая звезда остаётся за пределами области "процента толчка",  PHD2 увеличит силу толчка пока звезда не окажется внутри области.  Дальнейшие движения от этой точки до центральной позиции осуществляются с величиной, указанной в параметре "шаг толчка".  Обеспечивать хорошее гидирование непросто, особенно во время толчков.  АО продолжает вносить корректировки, поэтому долгие толчки непрерывно компенсируются корректировками  AO.

    При установленном параметре 'Толчок при подвижке' PHD2 будет осуществлять толчок всякий раз при получении команды подвижки и таким образом подвигать опорную звезду ближе к центральному положению АО.  Включение и выключение гидирования AO происходит независимо от параметра 'Включить гидирующие импульсы' во вкладке гидирования.  Таким образом, включать и выключать гидирующие импульсы на монтировку и АО можно независимо.  То же относится и к параметру 'Сбросить калибровку AO', который не влияет на калибровку монтировки. 'Компенсация люфта монтировки' позволяет исправлять люфт после толчка, что может ускорить проведение подвижек, но применима только при небольшом люфте склонения. Следует измерить аеличину люфта Помощником Гидирования, чтобы получить советы по его исправлению.

    При использовании адаптивной оптики вкладка 'Алгоритмы' покажет варианты управления наклонным элементом AO.



    Поскольку устройство АО не передвигает тяжёлое оборудование, можно выбирать более агрессивные параметры алгоритма гидирования.  По умолчанию задан 'Гистерезис', что даёт возможность легко управлять демпфированием и агрессивностью.
    .  В этом случае каждая поправка основывается только на последнем кадре и осуществляется100% коррекция последнего отклонения.  При использовании другого алгоритма следует начать с высокого уровня агрессивности, возможно 100%.   Другие параметры гидирования не показываются для АО, поскольку они не используются для управления устройством.

    В настоящее время поворотное устройство управляется только одним параметром.  Флажок "Реверс" может быть использовано для оптических систем, которые переворачивают изображение из-за нечётного количества зеркал.  Направление и величина поворота определяются данными калибровки, при этом PHD2 использует стандарт ASCOM:  "положение поворотного устройства выражается углом от 0 до 360 градусов исключительно против часовой стрелки относительно неба."
